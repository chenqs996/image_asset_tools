# 美术素材处理工具 V2 需求文档（游戏开发向）

## 1. 文档目标

定义 V2 版本的功能范围、交互方案、输出规范与验收标准，服务于游戏开发工作流（首发面向 Godot 4.x）。

---

## 2. 版本定位与范围

### 2.1 版本定位

V2 聚焦“可直接喂给引擎”的资源产出能力，覆盖：

1. 图集打包
2. 动画序列导出
3. 游戏 UI 专用切图增强
4. 引擎对接（首发仅 Godot 4.x）

### 2.2 平台范围

- 引擎：**Godot 4.x**（首发）
- 非目标平台：Godot 3.x、Unity、Cocos（后续版本评估）

### 2.3 非功能优先级

优先级：**导出稳定性与可追溯性 > 速度 > 极致压缩率**

---

## 3. 与现有交互风格的一致性要求

基于现有交互（顶层 tab + 局部子 tab + 顶部共享操作区）保持一致：

1. 顶部 `导入 / 清空 / 导出` 与素材选择保持共享入口
2. 新增能力优先通过“现有处理页内模块化卡片 + 配置区”接入
3. 允许对导出交互做结构升级（用户已确认可重构）
4. 保持“预览先行，导出确认后执行”的节奏

---

## 4. 功能需求明细

## 4.1 图集打包（Atlas Packer）

### 4.1.1 目标

将多张素材打包为一个或多个图集，并输出可追溯 metadata，供 Godot 4.x 读取。

### 4.1.2 首发交付（已确认）

- **首发必做输出：PNG + JSON（自定义 metadata）**
- 以下为后续排期（V2.x）：
  - PNG + Godot AtlasTexture 资源描述（.tres/.res）
  - 仅 PNG（无 metadata）

### 4.1.3 打包策略

- 默认策略：**平衡策略**（图集数量与空白率折中）
- 默认最大图集尺寸：**自动估算并提示**（用户可手动改）
- 推荐可选项（需实现）：
  - padding
  - extrude
  - 是否允许旋转
  - 是否强制 power-of-two

### 4.1.4 输出 JSON（首发）

至少包含：

- 图集信息：尺寸、像素格式、页索引
- Frame 信息：`name`, `x`, `y`, `w`, `h`, `rotated`
- 原始尺寸信息：`source_w`, `source_h`
- trim 信息：`trimmed`, `offset_x`, `offset_y`
- 可选锚点信息：`pivot_x`, `pivot_y`

### 4.1.5 验收标准

- 同一输入在相同配置下可稳定复现（frame 坐标一致）
- 导出日志可追踪：输入数量、失败项、分页结果
- 出现异常素材时不中断整批导出（记录失败并继续）

---

## 4.2 动画序列导出

### 4.2.1 目标

把时间线/序列结果导出为可直接导入 Godot 的动画资源输入。

### 4.2.2 首发交付（已确认）

全部纳入 V2：

1. 逐帧图片序列 + 动画描述 JSON
2. SpriteSheet + 帧索引 JSON
3. AnimationPlayer 可导入描述（JSON 形式，`.tres` 后补）

### 4.2.3 锚点与对齐

- 锚点模式：
  - 中心
  - 底边中心
  - 自定义
- 自定义输入：**支持归一化坐标与像素坐标，且可切换**

### 4.2.4 动画描述 JSON（建议字段）

- `animation_name`
- `fps`
- `loop`
- `frames[]`:
  - `frame_name` 或 `atlas_frame_id`
  - `duration_ms`（可选）
  - `pivot`
  - `offset`

### 4.2.5 验收标准

- 时间线顺序与导出顺序一致
- FPS/Loop 配置可正确写入
- 锚点导出值与 UI 设置一致

---

## 4.3 游戏 UI 专用切图增强

### 4.3.1 目标

提升 UI 资产生产效率，减少引擎内手工切图配置。

### 4.3.2 首发交付（已确认）

1. 九宫格切图 + margin 参数导出
2. 多分辨率倍率导出（1x / 1.5x / 2x）
3. 状态图自动拆分（normal/hover/pressed/disabled）

### 4.3.3 状态图识别规则（已确认）

默认按文件名后缀识别：

- `_normal`
- `_hover`
- `_pressed`
- `_disabled`

### 4.3.4 验收标准

- 状态识别失败的素材需给出明确提示（命名不匹配）
- 九宫格参数导出与预览一致
- 多倍率导出文件命名稳定（可复现）

---

## 4.4 Godot 4.x 对接

### 4.4.1 目标

输出 Godot 友好资源包（纹理 + metadata），降低引擎内二次配置成本。

### 4.4.2 输出格式（已确认）

- 首发：**JSON**（供 Godot 脚本/插件读取）
- `.tres/.res` 后续排期

### 4.4.3 资源包建议结构

- `textures/`：图集或逐帧纹理
- `metadata/`：atlas / animation / ui-slice JSON
- `manifest.json`：版本、导出时间、配置摘要、统计信息

### 4.4.4 验收标准

- Godot 侧读取 JSON 后可定位纹理区域与动画序列
- manifest 信息完整可审计
- 导出失败项可在日志和 manifest 中定位

---

## 5. 导出交互重构需求（V2）

允许重构导出交互，建议升级为“导出模板 + 任务参数”结构：

1. 选择导出模板
   - Atlas
   - Animation Sequence
   - UI Slice
   - Godot Package
2. 模板参数面板
   - 显示模板专属配置项
3. 预览与校验
   - 导出前进行配置校验与风险提示
4. 执行与摘要
   - 成功数/失败数/耗时/产物清单

---

## 6. 里程碑建议

### M1（基础可用）

- Atlas：PNG+JSON
- 动画：序列 + JSON
- UI：九宫格 + 多倍率
- Godot：基础 metadata JSON + manifest

### M2（体验增强）

- 动画：SpriteSheet + 帧索引 JSON
- UI：状态图自动拆分
- 导出交互重构（模板化）
- 日志与错误分类增强

### M3（扩展对接）

- AnimationPlayer 导入描述增强
- Godot 侧示例脚本
- `.tres/.res` 输出（如评估通过）

---

## 7. 验收测试集规模

用户确认目标：**小规模批次 < 200 张/次**

验收应覆盖：

1. 单图导出
2. 多图同规格批量
3. 多规格混合输入
4. 命名异常输入
5. 部分损坏素材输入（容错）

---

## 8. 已确认项汇总（决策记录）

1. 仅 Godot 4.x
2. 图集首发优先 PNG+JSON（其余后补）
3. 图集策略：平衡策略
4. 图集尺寸：自动估算并提示
5. 动画导出形态：序列+JSON、SpriteSheet+JSON、AnimationPlayer 描述 JSON
6. 锚点：中心/底边中心/自定义
7. 自定义锚点输入：归一化与像素坐标双支持
8. UI 状态识别：文件名后缀 `_normal/_hover/_pressed/_disabled`
9. 允许重构导出交互
10. 非功能优先级：稳定与可追溯优先

---

## 9. 非本版本范围（明确排除）

- 非 Godot 引擎的模板化导出（Unity/Cocos）
- Godot 3.x 兼容
- 云端协作与远程任务调度
- AI 自动分组命名（可后续评估）
