# 美术素材处理工具需求文档

- 文档版本：v1.1
- 创建日期：2026-06-28
- 面向阶段：MVP（首个可用版本）

## 1. 项目背景与目标

面向游戏美术素材制作流程，提供一个本地离线工具，减少素材切分、缩放、抠图、帧动画整理与批量导出的重复劳动，提高处理效率并保证命名与输出规范一致。

## 2. 用户角色

- 美术设计师：高频使用，关注预览效果与操作效率。
- 技术美术（TA）：关注参数可控、批处理一致性、命名规范。
- 客户端/资源管理人员：关注输出结构和可追溯性。

## 3. 使用场景

1. 将一张大图按固定尺寸切分成小块素材，先看切分线预览，确认后导出。
2. 将素材一键生成多种分辨率版本（例如 1x/0.5x/0.25x）。
3. 对单图先抠背景调参数，确认效果后把同参数应用到整批图片。
4. 导入一组帧图后进行动画预览，删除不需要帧并拖拽调整顺序。
5. 选中素材批量保存，按“统一前缀+递增序号”命名。

## 4. 功能需求

### 4.1 切分图片

#### 4.1.1 输入
- 支持格式：PNG/JPG/WebP/BMP（MVP）
- 单次选择：单张图片

#### 4.1.2 参数
- 切分计算模式（必选）
  - 指定单片分辨率：输入 `slice_width/slice_height`
  - 指定横竖数量：输入 `count_x/count_y`，自动计算单片尺寸
  - 线条提取：基于自动识别横纵切分线，或手动点击添加/调整切分线
- 起始偏移 `offset_x/offset_y`（可选，默认 0）
- 切分锚点（可选）
  - 左上（默认）
  - 中心（仅用于确定切分参考锚点，不影响编号顺序）
- 编号顺序策略（可选）
  - 按行遍历（左上到右下，默认）
  - 其他策略（后续扩展）
- 边缘处理策略（默认）：不足尺寸区域自动补透明，保证输出切片尺寸一致

#### 4.1.3 交互与行为
- 在原图上叠加网格线实时预览。
- 参数变化后预览自动刷新。
- 线条提取模式下支持自动识别切分线（首版优先策略：边缘检测 + 投影峰值），并允许手动点击/拖拽修正线条。
- 点击“确认切分”后生成切片列表（含缩略图与顺序编号）。
- 点击“保存”将切片输出到目标目录。

#### 4.1.4 验收标准
- 网格线位置与实际输出切片完全一致。
- 常规 4096×4096 图切分预览刷新时间 ≤ 300ms（参考硬件下）。
- 保存后的切片数量与预览统计一致。
- 使用默认边缘策略时，所有输出切片分辨率保持一致（边缘自动补透明）。
- 默认编号顺序与“按行遍历（左上到右下）”一致。

---

### 4.2 缩放图片

#### 4.2.1 输入
- 支持单图或多图批量输入。

#### 4.2.2 参数
- 缩放模式：按比例 / 按目标分辨率
- 比例列表（例如 `1, 0.5, 0.25`）
- 插值算法：最近邻、双线性、Lanczos（默认 Lanczos）
- 是否保持纵横比（默认是）
- 仅支持缩小（禁止放大）

#### 4.2.3 行为
- 生成多个分辨率版本并可预览对比。
- 批量模式下显示进度、成功/失败计数。

#### 4.2.4 验收标准
- 输出分辨率与配置一致。
- PNG 保持 Alpha 通道不丢失。

---

### 4.3 扣除背景（抠图）

#### 4.3.1 模式
- 单图调参模式（先试效果）
- 批量应用模式（复用已确认参数）

#### 4.3.2 输入
- 单图：用于参数调优
- 多图：批处理输入

#### 4.3.3 参数（首版）
- 抠图算法选择（首版固定三档，可扩展）
  - `AI通用`
  - `纯色色键`
  - `灰白方格专用`
- 前景检测阈值（低/中/高 或数值）
- 边缘平滑强度
- 去噪强度
- 羽化半径
- 边缘偏好（保留细节 / 去除毛边）
- 输出格式：PNG/BMP/WebP（批量输出格式统一）

#### 4.3.4 交互与行为
- 单图模式可实时预览“原图 / 抠图结果 / 差异对比”。
- 参数可保存为“当前配置”。
- 批量模式一键应用该配置，展示处理进度与失败日志。
- 批量处理后，允许对单张图片单独设置“例外参数”并重新处理。

#### 4.3.5 验收标准
- 单图预览与批量输出效果一致（同参数）。
- 批量失败不阻塞其他文件，失败项有明确原因记录。

---

### 4.4 帧动画处理

#### 4.4.1 输入
- 批量导入图片（按文件名默认排序）

#### 4.4.2 功能
- 动画预览（播放/暂停/循环）
- FPS 调节（例如 1~60）
- 删除选中帧
- 拖拽调整帧顺序
- 显示当前帧索引与总帧数

#### 4.4.3 验收标准
- 顺序调整后预览顺序立即更新。
- 删除帧后索引连续，播放无异常跳帧。

---

### 4.5 批量保存与命名规则

#### 4.5.1 选择与输出
- 支持在结果列表中多选素材后批量保存。
- 目标路径可配置。

#### 4.5.2 命名规则
- 统一前缀：`prefix`
- 递增序号：`001, 002, 003...`（位数可配，默认 3 位）
- 可选后缀：处理类型标签（如 `_slice`, `_bg`, `_scale_50`）

#### 4.5.3 冲突处理
- 文件存在时策略：跳过 / 覆盖 / 自动重命名（默认自动重命名）

#### 4.5.4 验收标准
- 命名连续且无重复。
- 实际输出文件名与预览列表一致。

## 5. 非功能需求

- 性能：
  - UI 操作保持流畅，长任务异步执行，不阻塞主界面。
- 稳定性：
  - 批处理单文件失败可恢复，不导致整批中断。
- 可用性：
  - 常用参数具备默认值、最近一次配置记忆。
- 兼容性：
  - 首版目标平台：Linux（优先），后续可扩展 Windows/macOS。
- 可观测性：
  - 错误日志可导出，便于定位失败原因。

## 6. 数据与配置需求

- 最近使用路径记录
- 最近参数配置记录
- 用户预设（例如常用抠图参数）
- 配置持久化格式：JSON（MVP）

## 7. 约束与边界（MVP）

- 暂不支持 PSD、EXR 等复杂格式。
- 暂不内置 AI 训练，仅调用已有抠图模型。
- 暂不支持多人协作与云端素材库。

## 8. 已确认规则（2026-06-28）

- 输入格式新增 BMP。
- 批量输出格式统一，支持在 PNG/BMP/WebP 中选择一种。
- 切分边缘不足尺寸时自动补透明。
- 支持切分顺序方向：左上到右下（默认）/中心。
- 文件名无需坐标，但必须包含顺序编号（默认按左上到右下编号）。
- 切分尺寸支持三种计算方式：
  1) 指定分辨率；
  2) 指定横竖数量反算单片尺寸；
  3) 自动/手动切分线提取。
- 缩放仅支持缩小，不支持放大。
- 抠图背景重点场景为纯色和仿透明灰白方格背景。
- 抠图支持算法列表选择；批量后允许少量文件例外参数；边缘偏好为可配置项。

## 9. 已冻结决议（v1.1，2026-06-28）

- “中心”用于切分参考锚点，不参与编号顺序定义。
- 编号顺序策略可选，默认“按行遍历（左上到右下）”。
- 自动切分线识别首版采用“边缘检测 + 投影峰值”策略。
- 抠图算法首版固定三档：`AI通用` + `纯色色键` + `灰白方格专用`。
- 例外参数作用范围为“仅单张图片覆盖”。
- 缩放功能当前只实现缩小，不支持放大。

## 10. 里程碑建议

1. M1：基础框架 + 图片加载 + 切分预览与保存
2. M2：缩放批处理 + 批量命名保存
3. M3：单图抠图调参与批处理
4. M4：帧动画预览 + 删除 + 拖拽排序
5. M5：联调、性能优化、打包发布
